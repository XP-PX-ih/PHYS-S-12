Machine Building

My team included Mazen, Stephen, Helena, and myself. Our team's decided goal was to create a drawing machine that could recreate images. This section only includes my work on the end effector, for any who may wish to reuse it. For a full documentation of the project, see Stephen's Website.

To figure out how we could print in color with as few as possible, I looked up the different color theories. I decided upon CMYK: Cyan, Magenta, Yellow, and Black. This would allow for as many colors as possible with the fewest number of pens.

To make sure a servo could power my design, I did a little bit of physics, as shown below. I got the servo torque from this video: https://www.youtube.com/watch?v=PEcW4ja_glE

Load Torque is 0.644 N*m. Measured required torque for current design. Since the pivot is in the middle, and there exists gravitational torque on either side, I took half of the design as the mass and measured the gravitational torque on that. Assuming wood to be the material of choice, I found the weight of a piece of plywood, divided the weight by the area, and applied it to the chosen area of the design. The mass came out to be 0.2019kg. To quickly get an estimate on torque, I set the center of gravity in the middle of the horizontal axis. The estimated gravitational torque is 0.146 N*m. In theory, the servo motor should handle the weight just fine. I'm realizing now I forgot about depth, but the design calls for a small depth, so hopefully it makes a negligable impact on the needed torque.

I printed a smaller version to test my tightening mechanism: clamps. The clamps did not work and, in fact, snapped. Instead, I reused the tightening mechanism that the kit came from but in a smaller version. I redesigned the pencil attachement to instead hold a server, which would then hold the pen switching mechanism. It failed 3 times due to sizing errors, but I finally got it on the fourth.

I bought 3 edding 3000 markers - cyan, magenta, and yellow - and a black sharpie from Blick Art Supplies. I adjusted the pen diameters and lengths in the CAD and reprinted. After testing it with the servo, I found that it was unstable and would consistently fly off on the second turn. To try and fix this problem, I tapped two holes into the print just to the side of where the servo motor was placed. I then tied twill to connect the pen swapper to the wooden dowels that connected the servo attachment to the Y-axis arm. The reason I used string was because I didn't want to design and reprint a mechanism that would allow for the pen swapper to be connected to a stationary object while it rotated. After spending 40 minute trying to get it to work, I realized I literally built in a hole for the pen swapper to screw into the servo. After that, the entire problem was sovled.

I repurposed the UselessServo code and tested out both the pen swapping and the pen lifting functions. It worked, thank goodness. What's left for that is now fine tuning the rotations to match each marker color.


#include 
int switchServoPin = D6;
int liftpin = D7;

class UselessServo{
  Servo servo;
  int pos;
  int increment;
  int updateInterval;
  unsigned long lastUpdate;
  int startPos;
  int endPos;

  public:
  UselessServo(int interval, int start, int target){
    updateInterval = interval;
    increment = -1;
    startPos = start-1;
    pos = startPos;
    endPos = target;
  }

  void Attach(int pin){
    servo.attach(pin);
    servo.write(startPos);
  }

  void Detach(){
    servo.detach();
  }

  void toTarget(){
    servo.write(endPos);
  }

  void Return(){
    servo.write(startPos);
  }
};

UselessServo pens(0.5, 180, 0);
UselessServo lift(0.5, 180, 90);

void setup() {
  pens.Attach(switchServoPin);
  lift.Attach(liftpin);
  Serial.begin(9600);
}

void loop() {
  // teatime.locsl
    pens.toTarget();
    delay(3000);
    pens.Return();
    delay(3000);
    lift.toTarget();
    delay(1500);
    lift.Return();
}
